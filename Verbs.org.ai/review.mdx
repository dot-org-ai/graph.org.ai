# Review

Examining or assessing materials, documents, work, or performance to verify quality, accuracy, compliance, and suitability for intended use.

## O*NET Context

Work activity focused on examining information, documents, or work products to check for correctness, quality, completeness, and adherence to standards or policies.

## Semantic Forms

### Action (Base Form)
```typescript
await $.QA.review({ content: document, criteria: ['accuracy', 'completeness', 'compliance'] })
```

### Predicate (Relationship)
```typescript
$.Manager.reviews.Report
$.Auditor.reviews.Process
```

### Event (Past Tense)
```typescript
on($.Document.reviewed, async (event) => {
  await $.Status.update({ approval: event.approved })
  await $.Archive.store({ review: event })
})
```

### Activity (Ongoing Process)
```typescript
const activeReview = $.QualityControl.reviewing.Batch
console.log(`Currently reviewing: ${activeReview.itemCount} items`)
```

## Usage Examples

**Quality Assurance:**
- Reviewing code for quality and best practices
- Examining products for defects or inconsistencies
- Checking documents for accuracy and completeness
- Verifying compliance with standards or regulations

**Performance Management:**
- Reviewing employee work and performance
- Assessing project deliverables and outcomes
- Evaluating proposal or application submissions
- Examining test results and data quality

**Administrative Oversight:**
- Reviewing budgets and expenditures
- Examining contracts before execution
- Checking reports for accuracy and clarity
- Assessing training materials or procedures
