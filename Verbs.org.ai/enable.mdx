# Enable

Making something operational, available, or capable of functioning.

## Schema.org Context

A schema.org Action representing the activation or provision of access to resources, features, or capabilities. Complementary to disable and essential in system administration and access control.

## Semantic Forms

### Action (Base Form)
```typescript
await $.Feature.enable({ user: customer, timestamp: now(), tier: 'premium' })
```

### Predicate (Relationship)
```typescript
$.Administrator.enables.Feature
$.System.enables.Permission
$.Service.enables.Capability
```

### Event (Past Tense)
```typescript
on($.Account.enabled, async (event) => {
  await $.Notification.send({ target: event.accountOwner, message: 'Account activated' })
  await $.Service.provision({ accountId: event.accountId, features: event.grantedFeatures })
})
```

### Activity (Ongoing Process)
```typescript
const enablingPhase = $.System.enabling.NewFeatures
console.log(`Activating ${enablingPhase.featureCount} new features for users`)
```

### Role (Actor)
```typescript
const enabler: $.Enabler = await $.findAgent({ permissions: 'system_admin' })
await enabler.grantAccess(userId, resource, level)
```

### Concept (Abstraction)
```typescript
const enabling: $.Enabling = {
```

### Inverse (Passive)
```typescript

```

## Usage Examples

**Account Management:**
- System enables account after verification
- Administrator enables user access
- Service enables premium features for subscriber

**Feature Activation:**
- Developer enables experimental feature
- Product manager enables feature for beta group
- System enables auto-renewal on account

**Business Rules:**
- Enabling requires valid authorization
- Enabling triggers provisioning workflows
- Enabling can be time-limited or permanent
