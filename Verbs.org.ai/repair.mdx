## Context

In supply chain and service operations, repair occurs when equipment or products are restored to functional condition:

- **Maintenance repair**: Regular maintenance-driven repairs
- **Warranty repair**: Repairing under warranty coverage
- **Damage repair**: Restoring damaged goods
- **Component replacement**: Replacing faulty parts
- **Functionality restoration**: Returning to operational status
- **Quality assurance**: Testing after repair completion

## Implementation Example

```typescript
import { $ } from 'sdk.do'

// Record repair operation
const repairEvent = await $.Event.create({
  type: 'repairing',
  bizStep: 'urn:epcglobal:cbv:bizstep:repairing',
  epcList: ['urn:epc:id:sgtin:...'],
  readPoint: 'urn:epc:id:sgln:service-center-id',
  bizLocation: 'urn:epc:id:sgln:repair-facility-id',
  timestamp: new Date().toISOString(),
  action: 'OBSERVE',
  disposition: 'in_progress'
})

// Create repair work order
const repairOrder = await $.WorkOrder.create({
  type: 'repair',
  productId: 'device-sku-123',
  issueDescription: 'Display malfunction',
  assignedTechnician: 'tech-id-001',
  priority: 'high',
  estimatedCompletionDate: new Date(Date.now() + 2 * 24 * 60 * 60 * 1000)
})

// Update product status
const repairedProduct = await $.Product.repair({
  productId: 'device-id',
  repairId: repairOrder.id,
  technician: 'tech-id-001',
  partsReplaced: ['display-unit', 'power-cord'],
  completionDate: new Date().toISOString()
})

// Emit repair completion notification
on($.Equipment.repaired, async (event) => {
  await send($.Notification.send, {
    type: 'repair_complete',
    equipmentId: event.equipmentId,
    repairId: event.repairId,
    repairedBy: event.repairedBy
  })
})
```

## Related Verbs

- **install**: Putting repaired items into service
- **inspect**: Examining for quality before/after repair
- **decommission**: Removing items that cannot be repaired
- **replace**: Substituting irreparable items with new ones
