# Estimate

Judging, calculating, or approximating values, quantities, costs, time periods, or outcomes based on available information.

## O*NET Context

Work activity involving making judgments or calculations about quantities, values, or outcomes. Common in construction, project management, accounting, engineering, and sales roles where accurate estimations directly impact business operations.

## Semantic Forms

### Action (Base Form)
```typescript
await $.Contractor.estimate({ project: 'renovation', scope: buildingDetails })
await $.Analyst.estimate($.Revenue, { period: 'Q4', method: 'historical-trend' })
```

```typescript
on($.Bid.estimated, async (event) => {
```

## Usage Examples

**Construction and Trades:**
- Contractor estimates project costs and timeline
- Builder estimates material quantities
- Electrician estimates labor and materials
- Plumber estimates repair costs

**Business and Finance:**
- Financial analyst estimates revenue projections
- Budget manager estimates departmental costs
- Sales manager estimates deal probability and close timeline
- CFO estimates cash flow requirements

**Project Management:**
- Project manager estimates task duration
- Resource manager estimates staffing needs
- Risk analyst estimates probability and impact
- Scope manager estimates deliverables and effort

**Business Rules:**
- Estimates should use consistent methodologies
- Estimates require documented assumptions
- Estimates should include confidence levels
- Estimate accuracy should be tracked and reviewed
