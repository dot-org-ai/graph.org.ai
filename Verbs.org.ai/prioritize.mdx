# Prioritize

Determining the order of importance or sequence in which tasks, items, or actions should be addressed based on criteria.

## O*NET Context

Work activity focused on ranking or sequencing work based on urgency, importance, resources, or organizational objectives.

## Semantic Forms

### Action (Base Form)
```typescript
await $.Manager.prioritize({ items: taskList, criteria: ['urgency', 'impact', 'resources'] })
```

### Predicate (Relationship)
```typescript
$.Team.prioritizes.Tasks
$.Project.prioritizes.Resources
```

### Event (Past Tense)
```typescript
on($.Tasks.prioritized, async (event) => {
  await $.Team.communicate({ sequence: event.sequence })
  await $.Timeline.update({ order: event.priorityOrder })
})
```

### Activity (Ongoing Process)
```typescript
const activePrioritization = $.Manager.prioritizing.Workload
console.log(`Currently prioritizing: ${activePrioritization.itemCount} items`)
```

### Role (Actor)
```typescript
const prioritizer: $.Prioritizer = await $.findAgent({ expertise: 'resource-allocation' })
await prioritizer.rankItems(requests)
```

### Concept (Abstraction)
```typescript
const prioritization: $.Prioritization = {
  subjectId: workload.id,
  items: [],
  criteria: [],
  sequence: [],
  rationale: '',
  timestamp: now()
}
```

### Inverse (Passive)
```typescript
const managers = await db.related($.Tasks, $.prioritizedBy, $.Manager)
```
