# Revise

Making changes, corrections, or improvements to documents, plans, work products, or processes to enhance quality, accuracy, or effectiveness.

## O*NET Context

Work activity focused on editing and updating materials to fix errors, improve clarity, incorporate feedback, or adapt to changing requirements.

## Semantic Forms

### Action (Base Form)
```typescript
await $.Editor.revise({ document: manuscript, feedback: comments })
```

### Predicate (Relationship)
```typescript

```

### Event (Past Tense)

## Usage Examples

**Document Management:**
- Revising reports based on feedback
- Updating proposals with new information
- Editing manuscripts for clarity and accuracy
- Modifying policies to reflect changes

**Project Management:**
- Revising project plans based on progress
- Updating timelines and schedules
- Modifying scope based on stakeholder input
- Adjusting budgets and resource allocations

**Software Development:**
- Revising code based on review feedback
- Updating documentation to match implementation
- Modifying designs based on requirements changes
- Improving algorithms based on performance data
