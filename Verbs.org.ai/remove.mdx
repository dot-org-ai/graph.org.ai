# Remove

Taking away items from a location or active use.

## GS1 Context

Business step for formally removing items from inventory, storage locations, or active service. This operation applies to various scenarios including shelf removal, equipment decommissioning, or inventory write-offs.

## Semantic Forms

### Action (Base Form)
```typescript
await $.InventoryItem.remove({ reason: 'expired', timestamp: now(), location: 'Aisle-B' })
```

### Predicate (Relationship)
```typescript
$.Warehouse.removes.ExpiredGoods
$.Store.removes.DiscontinuedItems
```

### Event (Past Tense)
```typescript
on($.Item.removed, async (event) => {
  await $.Inventory.decrease({ sku: event.sku, quantity: event.quantity })
  await $.Accounting.recordWriteOff({ itemId: event.id, reason: event.reason })
})
```

### Activity (Ongoing Process)
```typescript
const disposal = $.Warehouse.removing.DamagedInventory
console.log(`Currently removing: ${disposal.count} items`)
```

### Role (Actor)
```typescript
const remover: $.Remover = await $.findAgent({ permissions: 'inventory-disposal' })
await remover.processRemoval(item)
```

### Concept (Abstraction)
```typescript
const removal: $.Removal = {
  itemId: item.id,
  timestamp: now(),
  remover: user.id,
  reason: 'quality-issue',
  location: location,
  disposalMethod: 'recycling'
}
```

### Inverse (Passive)
```typescript
const removers = await db.related($.Item, $.removedBy, $.QualityTeam)
```

## Usage Examples

**Inventory Management:**
- Removing expired items from shelves
- Removing damaged goods from inventory
- Removing discontinued products
- Removing recalled items from circulation

**Business Rules:**
- Removal must be documented with reason and timestamp
- Removed items must be deducted from inventory
- High-value removals require approval
- Removal triggers accounting adjustments
